\section{Approches avancees d'explicabilite : Stacking et XStacking}
\label{sec:cas4}

Cette section explore des techniques d'ensemble learning avancees (Stacking, XStacking) en mettant l'accent sur leur explicabilite. Le Stacking combine les predictions de plusieurs modeles de base via un meta-apprenant, mais pose un defi majeur : \textbf{la perte de tracabilite}. Nous proposons le paradigme \textbf{XStacking} qui enrichit le Stacking avec des valeurs SHAP pour restaurer l'explicabilite end-to-end.

\subsection{Stacking conventionnel avec validation croisee}
\label{ssec:cas4-stacking}

\subsubsection{Principe du Stacking}

Le \textbf{Stacking} (ou \textit{Stacked Generalization}, Wolpert 1992) est une methode d'ensemble learning a deux niveaux :
\begin{enumerate}
    \item \textbf{Niveau 0 (modeles de base)} : Un ensemble de $M$ modeles heterogenes $\{h_1, h_2, \ldots, h_M\}$ est entraine sur les donnees d'entrainement. Pour eviter le surapprentissage, on utilise la validation croisee stratifiee (K-Fold) pour generer des predictions \textit{Out-Of-Fold} (OOF) : chaque prediction $\hat{y}_i^{(m)}$ est produite par un modele $h_m$ qui n'a pas vu l'observation $(x_i, y_i)$ lors de l'entrainement.
    
    \item \textbf{Niveau 1 (meta-apprenant)} : Un meta-apprenant $g$ est entraine sur un nouveau dataset constitue des predictions OOF : $\mathcal{D}_{meta} = \{([\hat{y}_i^{(1)}, \ldots, \hat{y}_i^{(M)}], y_i)\}_{i=1}^N$. Le meta-apprenant apprend a combiner optimalement les predictions des modeles de base.
\end{enumerate}

La prediction finale pour une nouvelle observation $x$ est :
\begin{equation}
    \hat{y} = g(h_1(x), h_2(x), \ldots, h_M(x))
\end{equation}

\subsubsection{Configuration des modeles de base}

Le succes du Stacking repose sur la \textbf{diversite} des modeles de base. Nous avons selectionne 5 modeles couvrant differents paradigmes d'apprentissage pour maximiser la complementarite :

\begin{itemize}
    \item \textbf{KNN} : Apprentissage par proximite locale, non-parametrique
    \item \textbf{Extra Trees} : Ensemble avec splits aleatoires (randomisation accrue)
    \item \textbf{Random Forest (130 arbres)} : Ensemble d'arbres par Bagging (reduction de variance)
    \item \textbf{XGBoost (150 arbres)} : Gradient Boosting optimise (correction des erreurs residuelles)
    \item \textbf{Naive Bayes} : Modele probabiliste (hypothese d'independance)
\end{itemize}

Cette diversite garantit que les modeles ont des biais et des variances differents, condition necessaire pour que le Stacking ameliore la performance (theoreme de Wolpert).

\begin{lstlisting}[style=reportPython, caption={Configuration des modeles de base avec principe de complementarite}]
from sklearn.ensemble import RandomForestClassifier, ExtraTreesClassifier
from sklearn.neighbors import KNeighborsClassifier
from sklearn.naive_bayes import GaussianNB
from xgboost import XGBClassifier

base_learners = [
    ('knn', KNeighborsClassifier()),
    
    ('et', ExtraTreesClassifier()),
    
    ('rf', RandomForestClassifier(
        n_estimators=130,
        max_depth=11,
        min_samples_split=3,
        min_samples_leaf=2,
        max_features='sqrt',
        criterion='entropy',
        bootstrap=True,
        random_state=42,
        n_jobs=-1
    )),
    
    ('xgb', XGBClassifier(
        n_estimators=150,
        max_depth=5,
        learning_rate=0.5,
        objective='binary:logistic'
    )),
    
    ('nb', GaussianNB(var_smoothing=1e-9))
]
\end{lstlisting}

\subsubsection{Construction du Stacking}

Le meta-apprenant choisi est une \textbf{Regression Logistique} pour sa simplicite et son explicabilite intrinseque. Ses coefficients $\beta_m$ indiquent directement l'importance de chaque modele de base dans la decision finale :

\begin{equation}
    P(y=1 | \hat{\mathbf{y}}) = \sigma\left(\beta_0 + \sum_{m=1}^M \beta_m \hat{y}^{(m)}\right)
\end{equation}

ou $\sigma$ est la fonction sigmoide et $\hat{y}^{(m)}$ la probabilite predite par le modele $m$.

\begin{lstlisting}[style=reportPython, caption={Construction du Stacking avec validation croisee}]
from sklearn.ensemble import StackingClassifier
from sklearn.linear_model import LogisticRegression

meta_learner = LogisticRegression(
    penalty='l2',
    C=1.0,
    solver='lbfgs',
    max_iter=1000,
    random_state=42
)

stacking_clf = StackingClassifier(
    estimators=base_learners,
    final_estimator=meta_learner,
    cv=6,
    n_jobs=-1,
    passthrough=False
)

stacking_clf.fit(X_train, y_train)
y_pred_stacking = stacking_clf.predict(X_test)
y_proba_stacking = stacking_clf.predict_proba(X_test)
\end{lstlisting}

\subsubsection{Resultats}

Le modele de Stacking atteint une \textbf{accuracy de 95.83\%} sur l'ensemble de test, egalant la performance du meilleur modele individuel (Extra Trees) tout en beneficiant d'une robustesse accrue grace a la diversite des modeles de base.

\subsection{XStacking : Stacking enrichi avec SHAP}
\label{ssec:cas4-xstacking}

\subsubsection{Motivation : Le probleme de la perte de tracabilite}

Bien que le Stacking ameliore les performances, il souffre d'un probleme majeur en contexte medical : \textbf{la perte de tracabilite vers les variables cliniques originales}. Le meta-apprenant voit uniquement les probabilites agregees des modeles de base, sans acces aux variables cliniques (hemoglobine, creatinine, albumine, etc.). Il est donc impossible de repondre a la question : \textit{"Pourquoi l'hemoglobine de ce patient influence-t-elle le diagnostic final ?"}

\subsubsection{Paradigme XStacking}

\textbf{XStacking} (Explanation-augmented Stacking) resout ce probleme en enrichissant le dataset du meta-apprenant avec les \textbf{valeurs SHAP} des modeles de base. L'architecture devient :

\begin{equation}
    \mathcal{D}_{XStack} = \left[\underbrace{\hat{y}^{(1)}, \ldots, \hat{y}^{(M)}}_{\text{Predictions OOF}} \mid \underbrace{\Phi^{(1)}, \ldots, \Phi^{(M)}}_{\text{Valeurs SHAP}}\right]
\end{equation}

ou $\Phi^{(m)} \in \mathbb{R}^{p}$ contient les importances SHAP moyennes du modele $m$ pour les $p$ variables cliniques.

\subsubsection{Calcul des valeurs SHAP}

Pour chaque modele de base, nous calculons les valeurs SHAP qui decomposent la prediction en contributions de chaque variable clinique. La strategie de calcul depend du type de modele :

\begin{itemize}
    \item \textbf{Modeles arborescents} (RF, ET, XGBoost) : \textbf{TreeExplainer} (calcul exact en temps polynomial)
    \item \textbf{Autres modeles} (KNN, NB) : \textbf{KernelExplainer} (approximation par echantillonnage)
\end{itemize}

\begin{lstlisting}[style=reportPython, caption={Calcul des valeurs SHAP pour les modeles de base}]
import shap

shap_values_dict = {}
X_sample = X_train[:100]

for name, model in trained_models.items():
    if name in ['rf', 'et', 'xgb']:
        # TreeExplainer pour modeles arborescents
        explainer = shap.TreeExplainer(model)
        shap_values = explainer.shap_values(X_sample)
        
        # Gestion du format de sortie
        if isinstance(shap_values, list):
            shap_values = shap_values[1]  # Classe positive
        if len(shap_values.shape) == 3:
            shap_values = shap_values[:, :, 1]
        
        shap_values_dict[name] = shap_values
        
    elif name == 'nb':
        # KernelExplainer pour Naive Bayes
        explainer = shap.KernelExplainer(
            model.predict_proba, 
            shap.sample(X_train, 100)
        )
        shap_values = explainer.shap_values(X_sample, nsamples=100)
        
        if isinstance(shap_values, list):
            shap_values = shap_values[1]
        
        shap_values_dict[name] = shap_values
\end{lstlisting}

On observe une \textbf{coherence remarquable} entre les modeles : l'\textbf{hemoglobine} est systematiquement la variable la plus importante, suivie de la \textbf{gravite specifique} et de l'\textbf{albumine}. Cette coherence inter-modeles valide la robustesse des insights cliniques et confirme que les modeles capturent les memes patterns physiologiques sous-jacents.

    \item \textbf{Gravite specifique < 1.015} : Perte de capacite de concentration renale
    \item \textbf{Albumine (presence)} : Proteinurie, dysfonction glomerulaire
    \item \textbf{Creatinine serique > 2.4 mg/dL} : Marqueur standard de la fonction renale
\end{itemize}

\subsubsection{Construction du dataset XStacking}

Le dataset XStacking est construit en concatenant les predictions OOF avec les importances SHAP agregees :

\begin{lstlisting}[style=reportPython, caption={Construction du dataset XStacking}]
# Predictions OOF des modeles de base via cross_val_predict
from sklearn.model_selection import cross_val_predict, StratifiedKFold

cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
oof_predictions = np.zeros((X_train.shape[0], len(base_learners)))

for idx, (name, model) in enumerate(base_learners):
    oof_pred = cross_val_predict(
        model, X_train, y_train, 
        cv=cv, 
        method='predict_proba',
        n_jobs=-1
    )
    oof_predictions[:, idx] = oof_pred[:, 1]

# Calcul des importances SHAP moyennes
shap_importances = []
for name, shap_vals in shap_values_dict.items():
    if shap_vals is not None:
        mean_abs_shap = np.mean(np.abs(shap_vals), axis=0)
        shap_importances.append(mean_abs_shap)

shap_importances = np.array(shap_importances)

# Concatenation : [Predictions OOF | Importances SHAP]
X_train_xstacking = np.hstack([
    oof_predictions,
    np.tile(shap_importances.flatten(), (X_train.shape[0], 1))
])
# Dimensionalite : 5 predictions + (4 modeles x 24 features) = 101 features
\end{lstlisting}

\subsubsection{Entrainement et evaluation}

Le meta-apprenant XStacking (egalement une Regression Logistique) est entraine sur ce dataset enrichi :

\begin{lstlisting}[style=reportPython, caption={Entrainement du meta-apprenant XStacking}]
meta_learner_xstacking = LogisticRegression(
    penalty='l2', C=1.0, solver='lbfgs', max_iter=1000, random_state=42
)

meta_learner_xstacking.fit(X_train_xstacking, y_train)

# Preparation des donnees de test
test_predictions = np.zeros((X_test.shape[0], len(base_learners)))
for idx, (name, model) in enumerate(trained_models.items()):
    test_predictions[:, idx] = model.predict_proba(X_test)[:, 1]

X_test_xstacking = np.hstack([
    test_predictions,
    np.tile(shap_importances.flatten(), (X_test.shape[0], 1))
])

y_pred_xstacking = meta_learner_xstacking.predict(X_test_xstacking)
xstacking_acc = accuracy_score(y_test, y_pred_xstacking)
\end{lstlisting}

XStacking atteint une accuracy de \textbf{95.83\%}, identique au Stacking conventionnel. Le modele maintient donc la performance tout en offrant une \textbf{tracabilite complete} jusqu'aux variables cliniques originales.

\subsection{Insights cliniques et validation}
\label{ssec:cas4-insights}

\subsubsection{Variables critiques identifiees}

L'analyse SHAP revele les variables les plus importantes pour la prediction de la CKD :

\begin{enumerate}
    \item \textbf{Hemoglobine} : Marqueur precoce de CKD. L'anemie resulte de la diminution de production d'erythropoietine (EPO) par les reins.
    
    \item \textbf{Gravite specifique} : Reflete la capacite de concentration renale. Une gravite specifique basse indique une perte de fonction tubulaire.
    
    \item \textbf{Albumine} : La proteinurie (albumine dans l'urine) indique une dysfonction glomerulaire, signe precoce de CKD.
    
    \item \textbf{Creatinine serique} : Marqueur standard de la fonction renale, utilise pour calculer le DFG (debit de filtration glomerulaire).
    
    \item \textbf{Hypertension} : Facteur de risque majeur. Relation bidirectionnelle : l'hypertension cause la CKD, et la CKD aggrave l'hypertension.
\end{enumerate}

\subsubsection{Validation medicale}

Ces resultats sont \textbf{coherents} avec :
\begin{itemize}
    \item Les guidelines KDIGO (Kidney Disease: Improving Global Outcomes) 2012
    \item La litterature recente sur la prediction CKD (Gogoi \& Valan 2025, Chouit et al. 2026)
    \item La pratique clinique standard (biomarqueurs de routine)
\end{itemize}

Cette coherence valide que les modeles capturent des patterns physiologiques reels et non des correlations spurieuses.

\subsection{Avantages et limites}
\label{ssec:cas4-avantages}

\subsubsection{Avantages du paradigme XStacking}

\paragraph{Pour les cliniciens :}
\begin{itemize}
    \item \textbf{Tracabilite complete} : Possibilite de remonter de la decision finale jusqu'aux variables cliniques
    \item \textbf{Confiance accrue} : Validation que le modele utilise des biomarqueurs connus
    \item \textbf{Detection d'anomalies} : Identification de patterns inhabituels
    \item \textbf{Aide a la decision} : Priorisation des examens complementaires
\end{itemize}

\paragraph{Pour les data scientists :}
\begin{itemize}
    \item \textbf{Debugging} : Identification des erreurs de prediction via l'analyse SHAP
    \item \textbf{Feature engineering} : Decouverte d'interactions non-lineaires
    \item \textbf{Validation} : Verification que le modele n'apprend pas de correlations spurieuses
    \item \textbf{Reglementaire} : Conformite avec les exigences d'explicabilite (RGPD, FDA)
\end{itemize}

\subsubsection{Limites}

\begin{enumerate}
    \item \textbf{Cout computationnel} : Le calcul SHAP pour KNN est tres lent (KernelExplainer)
    \item \textbf{Complexite du meta-apprenant} : 101 features difficiles a interpreter manuellement
    \item \textbf{Taille du dataset} : 400 patients = risque de surapprentissage pour XStacking
    \item \textbf{Stabilite} : Performance identique au Stacking (pas d'amelioration)
\end{enumerate}

\subsection{Synthese}

Le Stacking conventionnel atteint d'excellentes performances (95.83\% accuracy) en combinant 5 modeles de base diversifies via un meta-apprenant explicable (Regression Logistique). Cependant, il perd la tracabilite vers les variables cliniques originales. Le paradigme \textbf{XStacking} resout ce probleme en enrichissant le meta-apprenant avec les valeurs SHAP, permettant une explicabilite end-to-end tout en maintenant la meme performance. Ce resultat est optimal en contexte medical ou la confiance et la tracabilite sont primordiales.

L'analyse SHAP revele que l'\textbf{hemoglobine}, la \textbf{gravite specifique} et l'\textbf{albumine} sont les biomarqueurs les plus critiques pour la prediction de la CKD, en coherence parfaite avec les guidelines cliniques et la litterature recente (2023-2026). Ces resultats demontrent que les modeles d'ensemble, bien que complexes, peuvent etre rendus explicables et cliniquement validables via des techniques XAI appropriees.
